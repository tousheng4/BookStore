# BookStoreSample 详细设计

> 核心原则：详细设计不是重复总体设计，而是说明系统内部**到底是怎么实现的**。每个模块按以下模板描述：模块功能 → 输入数据 → 处理逻辑 → 输出结果。

---

## 1. 商品管理模块（StoreService + Products）

### 1.1 图书查询与筛选

**模块功能**：提供图书的分页查询、关键词搜索、分类筛选、价格区间筛选、排序能力。

**输入数据**：
- `keyword`：string，搜索关键词（匹配书名、作者、分类）
- `category`：string，分类名称
- `minPrice` / `maxPrice`：decimal，价格区间上下限
- `sortBy`：string，排序方式（newest/sales/price_asc/price_desc/rating/stock）
- `reviewedOnly`：bool，是否只看有评价的图书
- `page` / `pageSize`：int，分页参数

**处理逻辑**：
1. 构建基础查询 `dbContext.Products.AsNoTracking().Where(p => p.IsActive)`，只查询已上架图书
2. 按 keyword 用 `Contains` 匹配 Title、Author、Category 三个字段（SQLite LIKE 查询）
3. 按 category、minPrice、maxPrice 依次添加 `Where` 条件过滤
4. **价格排序特殊处理**：由于 SQLite 对 decimal 排序支持有限，当排序为 `price_asc/price_desc` 时，先 `ToListAsync()` 将数据加载到内存，再在 C# 中用 `OrderBy/OrderByDescending` 排序；其他排序在数据库层完成（`OrderByDescending(SalesCount)` 等）
5. 计算分页：`TotalPages = Ceiling(TotalCount / PageSize)`，并对 page 参数做 `Math.Min(Max(1, page), totalPages)` 边界保护
6. 返回 `PagedProductsResult(Items, TotalCount, Page, PageSize)`

**输出结果**：`PagedProductsResult`，包含当前页图书列表、总记录数、当前页码、页大小。

**关键设计点**：价格排序需要内存排序是一个 SQLite + EF Core 的 workaround，说明在注释中已标注。

---

### 1.2 图书详情与关联查询

**模块功能**：获取单本图书详情，以及同类推荐和最近浏览。

**输入数据**：
- `productId`：int，图书编号
- `userId`：string（可选），当前登录用户 ID
- `category`：string，图书分类（用于查同类推荐）

**处理逻辑**：

**GetProductAsync**：
- 直接通过 `FirstOrDefaultAsync(p => p.Id == id)` 查询，附加 `AsNoTracking()` 提升只读查询性能。

**GetRelatedProductsAsync（同类推荐）**：
- 查询条件：`Category == category && Id != productId && IsActive`
- 排序策略：先按 `SalesCount` 降序（热销优先），再按 `CreatedAt` 降序（新书其次）
- 限制返回 `take` 条（默认4条）

**GetProductsByIdsAsync（最近浏览）**：
- 接收一个 `IEnumerable<int>` 的 ID 集合
- 去重后按原始顺序排列（通过 `orderedIds.IndexOf(id)` 确定位置），而非数据库返回顺序
- 过滤条件：`IsActive`，保证下架图书不出现在浏览历史推荐中

**输出结果**：相关图书列表、最近浏览图书列表。

---

### 1.3 图书增删改

**模块功能**：后台管理员对图书的创建、更新、上下架、删除操作。

**输入数据**（UpdateProductAsync）：
- `BookProduct input`：包含 Title、Author、Publisher、Category、Price、Stock、Description、CoverUrl、IsActive
- `changedBy`：string，执行操作的管理员标识

**处理逻辑**：

**AddProductAsync**：
1. 设置 `CreatedAt = UpdatedAt = DateTime.UtcNow`
2. `Description` 和 `CoverUrl` 空值时初始化为空字符串（避免数据库 NULL 约束问题）
3. `dbContext.Products.Add(product)` → `SaveChangesAsync`

**UpdateProductAsync**：
1. 查询现有图书记录，不存在则返回 `false`
2. **库存变动检测**：比较 `oldStock != product.Stock`，若有变动则调用 `AddInventoryChangeLog` 记录库存流水（ChangeType = "AdminAdjustment"）
3. **低库存提醒**：若图书已上架且更新后 `Stock <= 5`，调用 `AddAdminNotificationAsync` 向所有管理员发送低库存通知
4. 更新所有字段后设置 `UpdatedAt = DateTime.UtcNow`

**DeleteProductAsync**：
- 直接调用 `dbContext.Products.Remove(product)` 物理删除（非软删除）

**输出结果**：布尔值表示操作是否成功。

## 2. 购物袋模块（Cart）

### 2.1 加入购物袋

**模块功能**：将图书加入当前用户的购物袋，支持数量累加。

**输入数据**：
- `userId`：string，用户 ID
- `productId`：int，图书 ID
- `quantity`：int，加入数量

**处理逻辑**：
1. **基础校验**：quantity <= 0 直接返回 false
2. **商品校验**：图书不存在或未上架（`!IsActive`）返回 false
3. **唯一键约束利用**：由于数据库对 `(UserId, ProductId)` 建了唯一索引，可以用 `FirstOrDefault` 查找是否已有记录
4. **数量累加逻辑**：
   - 若不存在记录：新建 `CartItem`，Quantity = quantity
   - 若存在：nextQuantity = existing.Quantity + quantity，再与 `product.Stock` 比较是否超库存
5. 超库存返回 false，否则保存

**输出结果**：布尔值表示是否成功加入。

**关键设计点**：利用数据库唯一约束保证同一用户同一图书只有一条购物袋记录，累加数量而非创建多条。

---

### 2.2 修改数量与移除

**模块功能**：更新购物袋中某图书的数量，或整体移除。

**输入数据**：
- `userId` + `productId`：定位记录
- `quantity`：新数量（Update时；Remove时无需传数量）

**处理逻辑**：
- **UpdateCartItemAsync**：
  - 查到记录且 quantity <= 0：执行物理删除（购物袋移除）
  - quantity > 0：与商品 `Product.Stock` 比较，不超库存则更新
- **RemoveCartItemAsync**：直接通过 `(UserId, ProductId)` 查到记录并删除

**输出结果**：无（void 方法）。

---

### 2.3 凑单助手

**模块功能**：根据当前购物车金额和已有优惠券，计算距最近可用优惠券门槛还差多少金额，并推荐可以凑单的图书。

**输入数据**：
- `userId`、当前购物车总金额 `cartTotal`、购物车中已有图书的 `productIds`

**处理逻辑**：
1. 查询用户已领取但未使用的优惠券ID列表（`claimedCouponIds`）
2. 查询所有满足条件的目标优惠券：未过生效期、未过期、`MinimumAmount > cartTotal`（当前金额未达到门槛的）
3. 按以下优先级排序目标优惠券：
   - 已领取的优先（`IsClaimed` 降序）
   - 门槛差距小的优先（`Gap` 升序）
   - 抵扣金额大的优先
4. 取排序后的第一个作为目标优惠券，计算金额差距 `gap = MinimumAmount - cartTotal`
5. 在购物车外的图书中（`!productIds.Contains(productId)`）查找凑单候选
6. 为每个候选图书计算 `RemainingGap = Math.Max(0, gap - product.Price)` 和是否达标 `ReachesTarget = product.Price >= gap`
7. 按达标优先、差额从小到大、价格差从小到大、销量从高到低排序

**输出结果**：`OrderBoosterResult(TargetCoupon, GapAmount, IsClaimed, Suggestions)`。

## 3. 订单模块（Order）

### 3.1 订单创建（CreateOrderAsync）——核心复杂事务

**模块功能**：将购物袋商品结算为正式订单，扣减库存，记录快照，处理优惠券，发放会员积分。

**输入数据**：
- `userId`、`receiverName`、`receiverPhone`、`shippingAddress`、`userCouponId`（可选）

**处理逻辑**：

```
[事务开始]
1. 获取用户购物袋全部 CartItems（含关联 Product）
2. 校验购物袋非空
3. 遍历每个 CartItem：商品不存在/未上架/数量超库存 → 回滚事务返回 null
4. 创建 Order 对象，Status = "新订单"，CreatedAt = UtcNow
5. 写入首条 OrderStatusHistory（FromStatus=null, ToStatus="新订单"）
6. 遍历 CartItems：
   a. 扣减 Product.Stock
   b. 每本图书创建 InventoryChangeLog（扣减记录，ChangeType="OrderCreated"）
   c. 若扣减后 Stock <= 5，发送低库存提醒给管理员
   d. 为 Order.Items 添加 OrderItem 快照（Title、Author、CoverUrl、UnitPrice 均快照下单时数据）
7. 计算 originalAmount = Sum(OrderItem.LineTotal)
8. 若选择了优惠券：
   a. 查询 UserCoupon 记录，校验：本人、未使用、未过期、门槛符合
   b. 校验失败则回滚事务
   c. 将 Coupon.Name 写入 Order.CouponName，DiscountAmount 写入 Order.DiscountAmount
9. 计算最终金额：TotalAmount = Math.Max(0, originalAmount - DiscountAmount)
10. 保存 Order 到数据库（此时 Order.Id 生成）
11. 补充 InventoryChangeLog 的 OrderId（此前为null）
12. 若使用了优惠券：更新 UserCoupon.UsedAt 和 OrderId
13. 调用 AwardMembershipAsync 发放会员积分
14. [事务提交]
15. 返回完整订单（含 Items 加载）
[事务结束]
```

**输出结果**：成功返回完整 `Order` 对象（含 Items 集合），失败返回 null。

**关键设计点**：
- 使用 `BeginTransactionAsync` 确保库存扣减、订单创建、优惠券使用、积分发放原子性
- OrderItem 保存图书快照（Title、Author、CoverUrl、UnitPrice），使得订单详情页展示与下单时一致，不受商品后续变更影响
- InventoryChangeLog.OrderId 在订单保存后才补充，因为保存前 Order.Id 尚不存在

---

### 3.2 订单状态流转

**模块功能**：管理订单在"新订单 → 已支付 → 已发货 → 已收货 → 已完成"链路上的推进，支持取消和退款。

**状态机**（`OrderStatuses.CanTransition`）：

```
新订单 → 已支付 / 已取消
已支付 → 已发货 / 已取消
已发货 → 已收货
已收货 → 已完成
```

**各操作处理逻辑**：

**CancelOrderAsync（取消）**：
- 用户侧：仅 `NewOrder` 可取消（`CanCustomerCancel`）
- 管理员侧：通过 `CanTransition` 判断
- 实现：开启事务 → `RestoreOrderStockAsync` 回滚库存 → 写入状态历史 → 提交事务

**PayOrderAsync（支付）**：
- 校验：`CanTransition(NewOrder, Paid)` 为 true
- 写入状态历史
- 发送"支付成功"通知给用户

**ShipOrderAsync（发货）**：
- 校验：状态可流转到 Shipped + 物流公司和单号均非空
- 保存 `TrackingCompany`、`TrackingNumber`、`ShippedAt`
- 写入状态历史
- 发送"订单已发货"通知给用户

**ConfirmReceiptAsync（确认收货）**：
- 校验：状态可流转到 Received
- 写入状态历史
- 发送"已确认收货"通知

**RequestRefundAsync（申请退款）**：
- 校验：订单状态为 Paid 或 Received 且 `RefundRequestedAt == null`
- 记录 `RefundRequestedAt` 和 `RefundReason`
- 发送"新的退款申请"通知给管理员

**ApproveRefundAsync / RejectRefundAsync（审核退款）**：
- 校验：`HasPendingRefund == true`（有申请但未审核）
- 写入状态历史（ToStatus = "已退款" 或保持原状态）
- 发送审核结果通知给用户

---

### 3.3 库存回滚（RestoreOrderStockAsync）

**模块功能**：取消订单时将扣减的库存回滚。

**处理逻辑**：
- 遍历 `Order.Items`，对每一条 OrderItem：
  - 查到对应 `Product`
  - `Product.Stock += item.Quantity`
  - 写入 `InventoryChangeLog`（ChangeType = "OrderCancelled"）
- 由于库存扣减和回滚都在事务内，确保一致性

## 4. 优惠券模块（Coupon）

### 4.1 优惠券领取

**模块功能**：用户从可领取列表中领取优惠券。

**输入数据**：`userId`、`couponId`

**处理逻辑**：
1. 校验优惠券：存在、启用 `IsActive`、在有效期 `StartsAt <= now <= EndsAt`
2. 校验领取约束：`UserId + CouponId` 唯一（数据库唯一索引），查询 `AnyAsync` 判断是否已领取
3. 两者任一不满足返回 false
4. 创建 `UserCoupon` 记录（`ClaimedAt = now`）

**输出结果**：布尔值。

---

### 4.2 优惠券使用（结算时）

**模块功能**：结算页根据订单金额筛选可用优惠券，订单创建时核销优惠券。

**可用优惠券判断**（`IsCouponUsable`）：
```
coupon.IsActive
&& coupon.StartsAt <= now
&& coupon.EndsAt >= now
&& coupon.MinimumAmount <= orderAmount（门槛）
&& coupon.DiscountAmount > 0
```

**结算页筛选**（`GetUsableCouponsAsync`）：
- 条件：本人 + 未使用 + 优惠券本身可用
- 排序：按 `DiscountAmount` 降序（最大抵扣优先），再按 `MinimumAmount` 升序（门槛低优先）

**订单核销**：
- 在 `CreateOrderAsync` 事务中，更新 `UserCoupon.UsedAt` 和 `OrderId`
- 订单保存 `CouponName`（快照）和 `DiscountAmount`（快照）

## 5. 评价与问答模块（Review & Question）

### 5.1 评价发布

**模块功能**：已购买用户对图书发布评分和文字评价。

**输入数据**：`userId`、`productId`、`rating`（1-5）、`content`（最多500字）、`imageUrl`（可选）

**处理逻辑**：
1. **购买资格校验**（`CanReviewProductAsync`）：
   - 查询 OrderItems，条件：`ProductId == productId && Order.UserId == userId && Order.Status ∈ {已支付, 已发货, 已收货, 已完成}`
   - 任一匹配则返回 true（可评价）
2. **评分范围校验**：1 ≤ rating ≤ 5
3. **唯一约束利用**：由于 `UserId + ProductId` 唯一索引，若已有评价则走更新逻辑（Update），无则新增（Add）
4. 更新时保留原 `CreatedAt`，仅更新 `UpdatedAt`
5. 发送"评价已发布"通知给用户

**追评**（`SaveReviewFollowUpAsync`）：
- 先查用户对本书的评价是否存在，不存在返回 false
- 更新 `FollowUpContent` 和 `FollowUpAt`
- 发送"追评已发布"通知

**晒图上传**（`SaveReviewImageAsync`）：
- 校验 `ContentType.StartsWith("image/")`
- 扩展名限制：`.jpg/.jpeg/.png/.webp/.gif`
- 保存路径：`wwwroot/uploads/reviews/{Guid}.{ext}`
- 返回相对路径 `/uploads/reviews/{filename}`

---

### 5.2 读者问答

**模块功能**：用户在图书详情页提交关于某图书的问题，管理员回答。

**提问**（`AskQuestionAsync`）：
- 校验：图书存在且已上架
- 提问内容非空，长度截断至500字
- 写入 `BookQuestion`，`Answer` 和 `AnsweredAt` 留空
- 发送"新读者提问"通知给管理员

**回答**（`AnswerQuestionAsync`）：
- 校验问题存在、回答内容非空
- 更新 `Answer`、`AnsweredBy`（管理员ID）、`AnsweredAt`
- 发送"提问已回答"通知给提问用户

## 6. 会员成长体系

### 6.1 积分与成长值发放（AwardMembershipAsync）

**模块功能**：订单完成后为用户发放积分和成长值，并判断是否触发等级升级。

**输入数据**：`userId`、`Order` 对象

**处理逻辑**：
1. 查询 `ApplicationUser`，不存在则直接返回
2. 计算本次成长值：`growthToAdd = Math.Floor(order.TotalAmount)`（每消费1元得1分）
3. 更新用户字段：`MemberPoints += growthToAdd`、`MemberGrowth += growthToAdd`
4. 若 `MembershipStartedAt == null`，设为当前时间（首次消费激活会员）
5. 根据新的 `MemberGrowth` 计算新等级：`BuildMemberLevel`
6. **等级升级判断**：比较新等级与 `oldLevel`，若变化则发送"等级升级"通知
7. 始终发送"积分到账"通知

**等级计算规则**（`BuildMemberLevel`）：
```
MemberGrowth >= 2000 → 黑钻会员
MemberGrowth >= 1000 → 金卡会员
MemberGrowth >= 300  → 银卡会员
else                 → 普通会员
```

---

### 6.2 会员中心数据

**模块功能**：聚合会员等级、积分、成长进度、最近积分记录。

**输入数据**：`userId`

**处理逻辑**：
1. 查询用户信息
2. 查询最近5笔有效订单作为"最近积分记录"
3. 计算升级进度：`ProgressPercent = (MemberGrowth - currentLevelStart) * 100 / (nextGrowth - currentLevelStart)`，用 `Math.Clamp` 限制在 0-100
4. 按等级返回不同权益列表（`GetMemberBenefits`）:
   - 普通会员及以上：积分回馈 + 会员提醒
   - 银卡会员及以上：优先凑单
   - 金卡会员及以上：专属推荐
   - 黑钻会员：高阶权益（预留）

## 7. AI 选书推荐（Book Assistant）

### 7.1 意图理解（BuildAssistantIntent）

**模块功能**：将用户自然语言输入解析为结构化意图。

**输入数据**：用户提问文本 `queryText`

**处理逻辑**：
- 对文本做大小写无关的分词（按空格、逗号、句号等分隔）
- 维护一个意图结构 `BookAssistantIntent(Categories, WantsCheap, WantsAdvanced, WantsPopular, WantsNew)`
- 通过关键词匹配推断用户偏好，例如：
  - "编程/开发/代码/程序" → Categories.Add("编程开发")
  - "便宜/低价/预算/入门" → WantsCheap = true
  - "进阶/深入/高级/专业" → WantsAdvanced = true
  - "热门/畅销/大家都买" → WantsPopular = true
  - "最新/新书/刚上架" → WantsNew = true

**输出结果**：`BookAssistantIntent` 结构体。

---

### 7.2 图书评分（ScoreAssistantProduct）

**模块功能**：对每本图书根据意图和多种信号计算推荐分数。

**输入数据**：图书对象、意图结构、用户偏好分类、销量、均分、评价数

**处理逻辑**（评分公式）：
```
score = 0
+ 关键词命中：每个匹配词 +8 分
+ 分类匹配意图：+34 分
+ 分类匹配用户偏好：+8 分
+ 低价需求满足（价格<=80）：+10 分
+ 进阶需求满足（描述含"进阶/深入"）：+10 分
+ 热销需求：销量 * 3 分
  否则：Min(销量, 20) 分
+ 新书需求满足（3个月内上架）：+12 分
+ 均分 * 2
+ Min(评价数, 10)
+ 有库存：+4 分；无库存：-20 分
```

**输出结果**：每本图书的综合推荐分，分数 > 0 才进入推荐列表。

---

### 7.3 推荐结果生成（AskBookAssistantAsync）

**模块功能**：整合意图理解、图书评分，返回推荐图书列表和推荐理由。

**处理逻辑**：
1. 加载全部已上架图书（含评价），查询全局销量数据
2. 获取用户偏好分类（`GetUserPreferredCategoriesAsync`，取用户历史订单中购买最多的3个分类）
3. 构建意图 → 对每本图书评分
4. 取分数 > 0 的图书按分数降序、均分降序、销量降序、上架时间降序排列
5. 若无匹配结果（用户提问无法解析），回退到"销量+均分+库存"基础分重新排序
6. 生成推荐理由（`BuildAssistantReason`）：从分类匹配、评分、销量、评价数中取最多3个理由拼接

**输出结果**：`BookAssistantResponse(Message, Suggestions)`，Suggestions 中每条包含 Id、Title、Author、Category、CoverUrl、Price、Reason。

## 8. 通知模块（Notification）

### 8.1 通知发送机制

**模块功能**：系统向用户或管理员发送事件驱动通知。

**通知类型**：Payment（支付）、Shipping（发货）、Receipt（确认收货）、Review（评价）、RefundRequest（退款申请）、RefundApproved/RefundRejected（退款审核）、LowStock（低库存）、MembershipPoints（积分到账）、MembershipLevelUp（等级升级）、BookQuestionAnswered（问答被回答）

**AddUserNotification**（写入通知）：
```
UserNotification {
    UserId = targetUserId,
    Title = title,
    Message = message,
    Type = type,
    LinkUrl = linkUrl,
    CreatedAt = UtcNow,
    ReadAt = null（默认未读）
}
```

**AddAdminNotificationAsync**（向所有管理员发通知）：
1. 查询 `AspNetRoles` 找到 Admin 角色
2. 查询所有 Admin 角色用户 ID
3. **去重检查**：若该类型通知已有未读通知指向同一链接，跳过（避免重复提醒）
4. 逐个写入通知

---

### 8.2 未读计数与已读管理

**GetUnreadNotificationCountAsync**：`CountAsync(UserId == userId && ReadAt == null)`

**MarkNotificationReadAsync**：找到通知后，`ReadAt ??= UtcNow`（仅首次标记时写入时间）

**MarkAllNotificationsReadAsync**：批量查询所有未读通知，遍历写入 `ReadAt`

## 9. 后台看板（Admin Dashboard）

### 9.1 经营数据聚合

**模块功能**：聚合图书、订单、销售、评价、优惠券等多维度经营数据。

**处理逻辑**：

| 指标 | 查询方式 |
|------|---------|
| 图书总数 | `Products.Count()` |
| 订单总数 | `Orders.Count()` |
| 待发货数 | `Orders.Count(Status == "已支付")` |
| 低库存图书数 | `Products.Count(IsActive && Stock <= 5)` |
| 评价总数 | `BookReviews.Count()` |
| 低分评价数 | `BookReviews.Count(Rating <= 2)` |
| 已支付销售额 | `Orders.Where(Status != NewOrder && Status != Cancelled).Sum(TotalAmount)` |
| 已完成销售额 | `Orders.Where(Status == Completed).Sum(TotalAmount)` |

---

### 9.2 图表数据计算

**模块功能**：生成近7日订单/销售额趋势图、分类销售占比图、畅销书排行图。

**近7日趋势**（`LoadChartsAsync`）：
- 时间范围：`DateTime.UtcNow.Date.AddDays(-6)` 至当天
- 每日统计：`OrderCount`（当日创建订单数）和 `SalesAmount`（当日有效订单销售额）
- `PaidRevenue` 的计算：状态为已支付/已发货/已收货/已完成的订单

**分类销售占比**：
- 从有效订单的 `OrderItems` 出发，按 `Product.Category` 分组
- 计算每类销售额 `Sum(LineTotal)`
- 取 Top 6，按销售额降序

**畅销书排行**：
- 从有效订单的 `OrderItems` 出发
- 按 `(ProductId, Title)` 分组，统计 `Sum(Quantity)`
- 取 Top 6

**图表百分比处理**（`WithPercent`）：
- 找到最大值 `max`
- 每项百分比 = `Max(4, Round(Value / max * 100))`（最小4%，避免0%时完全不可见）
